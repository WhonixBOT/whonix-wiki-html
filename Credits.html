Credits, Inspiration, Alternatives, Sources[edit]

Introduction[edit]

The success of the Whonix ™ platform is the sum total of several factors:

  • The solid foundation at its heart - the Debian GNU/Linux distribution.
  • The software provided by several other projects (like Tor and Qubes) and the tireless efforts of their many developers and contributors. [1]
  • Lessons learned from:
    • Expert advice provided in various Internet forums and attendance at software development events.
    • Previous failures and successes in the Whonix ™ development cycle.
    • The failures and successes of similar anonymity projects (like Tails).
  • Inspiration drawn from other projects, leading to the implementation of new or improved anonymity / security features.
  • Direct contributions by the Whonix ™ community.

Over the lifetime of the Whonix ™ project, development has been informed by thousands of external resources. For the reader's interest, a very small sample of older sources is listed here. Numerous updated sources can be found in developer documentation footnotes.

Similar Projects[edit]

There are a number of projects that share security and anonymity goals with Whonix ™. The reference list below captures a number of influential projects that have interesting proposals, security enhancements, desirable features and so on. [3] The intent is for any useful developments to be (potentially) ported to Whonix ™, as has occurred in the past.

Design Specifications[edit]

  • Tails [archive]: A live CD / USB distribution which is preconfigured to use Tor safely. [4]
    • PELD [archive]: Tails specification and implementation.
  • A Tor Virtual Machine Design and Implementation [archive]: Tor Project design documents.
  • Liberté Linux [archive]: A live CD based on hardened Gentoo. [5]
  • Qubes-Whonix ™ [archive]: This blog post provided the first hints on integrating Whonix ™ into Qubes. [6]
  • TorVM [archive] for Qubes: (qubes-tor [archive]).
    • tor-talk Review request: TorVM implementation in Qubes OS [archive].
    • qubes-devel TorVM implementation [archive].

Miscellaneous[edit]

  • VM basierende Anonymisierung [archive]: Unsophisticated German instructions on anonymizing VMs.
  • Onion Service Setup Guide for Newbies (Version .2) [archive]: A how-to on setting up a hidden (onion) service safely.
  • VMWare Through Tor [archive]: Forcing Tor to proxy connections from inside the VMWare browser application.
  • Broken links: Tor Onion Service Server in a box -- Tor Onion Service VM [archive] and "My Hidden Blog" - Hosting Tutorial [archive].

Networking and Firewalls[edit]

  • Tor and the DreamPlug [archive]: Similar networking / firewall design to Whonix ™.
  • Tor and OpenWRT [archive]: Similar networking / firewall design to Whonix ™.

Tor Bridges and Relays[edit]

  • Torouter [archive]: Alpha software to easily build a bridge or relay by default.
  • Running an OpenBSD Tor Relay: See Part 1 [archive] and Part 2 [archive] (unreviewed).

Transparent Proxies[edit]

  • JanusVM [archive]. [7]
  • TorVM [archive]: From the same authors as JanusVM, this project is also deprecated [archive].
  • Another TorVM [archive]: A contribution by unknown authors, with a site that imitates the official Tor one. [8]
  • Tor Project Transparent Proxy [archive]: Now outdated, but it offers: transparent anonymizing traffic for a specific user; local redirection through Tor; an anonymizing middlebox; and local redirection and an anonymizing middlebox. [9] [10]
  • Transocksifying Tor [archive]: This method to use a different firewall with DNT is now outdated, since it was only needed when Tor did not natively support TransPort.
  • TorVPN [archive]: The abandoned and unfinished predecessor to TorBOX. [11]
  • Old instructions from board.gulli.com [archive]: Complicated German instructions to utilize TransPort back when it was not natively supported by Tor.
  • Easy and secure anonymous internet usage [archive] by ra (an anonymous author). [12]
