Whonix ™ 16 Changelog[edit]

Whonix ™ 16 was released on September 11 and 12, 2021 for KVM and VirtualBox. [1] [2] Qubes-Whonix ™ 16 is also due for imminent release -- stay tuned for an upcoming announcement.

Significantly, Whonix ™ 16 is based on the Debian bullseye (Debian 11) distribution which was officially released [archive] on August 14, 2021 instead of Debian buster (Debian 10). The bullseye release has nearly 60,000 packages and around 72 per cent of them were updated. [3]

This means users have access to many new software packages in concert with existing packages. In addition, this release will serve as a development foundation for many exciting upcoming security enhancements such as Hardened Malloc Kicksecure ™ (HMK), Linux Kernel Runtime Guard (LKRG) and other items on the Whonix ™ Security Roadmap.

Major Changes[edit]

  • Ported the Debian base from Debian 10 buster to Debian 11 bullseye. [4]
  • Updated Monero GUI wallet (XMR) to version 0.17.2.3.
  • Updated Electrum Bitcoin (BTC) wallet to version 4.0.9 (Debian stable version).
  • Deprecated the binaries-freedom package; Electrum is now installed from packages.debian.org stable repository. [5]
  • Switched the tor package back from deb.torproject.org to packages.debian.org stable version. [6]
  • Enabled Debian Fast Track repository [archive] by default. [7]
  • Updated paths from /usr/lib to /usr/libexec where appropriate as per lintian Debian File System Hierarchy Standard (FHS) [archive].
  • VirtualBox version: this release utilizes VirtualBox guest additions version 6.1.26-dfsg-3 from fasttrack.debian.net. [8]

Known Issues[edit]

  • The desktop background image is currently the same for both Whonix-Gateway ™ and Whonix-Workstation ™. This will be corrected in a future release.

Whonix ™ 16 Updates[edit]

As Whonix ™ is a rolling distribution, users will benefit from regular small security and usability improvements, features and bug fixes as they enter the Whonix ™ stable repository. Those will be announced here.

The majority of the enhancements below also also apply to Qubes-Whonix ™. Exceptions include:

  • kloak
  • Linux Kernel Runtime Guard (LKRG)
  • tirdad [archive] (TCP ISN CPU Information Leak Protection)
  • Kernel Hardening through Kernel Boot Parameters [archive]
  • Strong Linux User Account Separation [archive] / Protection against Bruteforcing Linux User Account Passwords
  • apparmor-profile-everything [154] (AppArmor for everything: APT, systemd, init, all systemd units, all applications)
  • hardened-kernel patch and configuration [archive]

Many of these will be possible once the use of in-VM kernels is simplified and promoted in Qubes OS. [155] [156]
