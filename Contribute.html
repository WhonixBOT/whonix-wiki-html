Maintainer Responsibilities[edit]

The tasks of a maintainer include:

  • Answering questions in the forums.
  • Keeping the wiki up-to-date.
  • Testing of installation, build and upgrade instructions.
  • Following the issue tracker, squashing bugs and perhaps implementing new features relating to the maintained component.
  • Otherwise self-initiated (guess)work and taking care of the component in question.

This is a volunteer position.

Package Maintainer[edit]

Whonix â¢ provides a lot of Debian packages; see Overview [archive] for further information. Due to limited Whonix â¢ team resources, only a few packages are supported on Debian hosts. A lot of these packages are untested and unmaintained in any other Linux distribution.

Would you be interested in maintaining one or more packages for use beyond Whonix â¢? For example, in Debian, Ubuntu, Fedora or other distributions? Maintenance requirements would involve keeping up with code changes and releases, testing if these packages work outside of Whonix â¢, eventually reporting bugs or adding new features, and answering user support requests.

If interested, ambitious package maintainers could even seek to have one or more Whonix â¢ packages accepted into a mainstream Linux distribution.

This is a volunteer position.
